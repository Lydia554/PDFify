üì¢ Note for Developers: Handling the PDF Response

Our API returns the full PDF file data immediately in the response. Your application is responsible for processing this data to download or save the PDF file.

In Postman: After making the POST request, switch to the Body tab of the response, click Save Response (floppy disk icon), and save the file with a .pdf extension.

In your code: Handle the API response as a binary Blob (JavaScript) or save the raw response stream to disk (server-side), then trigger a file download or store the PDF as needed.

This approach gives you full control over how and when the PDF is saved or shown to your users.

üì¶ Packing Slip

fetch("https://pdf-api.portfolio.lidija-jokic.com/api/generate-packing-slip", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": "Bearer YOUR_API_KEY"
  },
  body: JSON.stringify({
    data: {
      orderId: "98765",
      customerName: "John Smith",
      shippingAddress: "1234 Maple Street, Springfield, USA",
      date: "2025-05-14",
      items: [
        { name: "Yoga Mat", sku: "YM-001", quantity: 1 },
        { name: "Dumbbells Set", sku: "DB-002", quantity: 2 },
        { name: "Water Bottle", sku: "WB-003", quantity: 3 }
      ]
    }
  })
})
.then(response => response.blob())
.then(blob => {
  const url = window.URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = "PackingSlip_98765.pdf";
  document.body.appendChild(a);
  a.click();
  a.remove();
  window.URL.revokeObjectURL(url);
})
.catch(console.error);
    
curl -X POST https://pdf-api.portfolio.lidija-jokic.com/api/generate-packing-slip \
-H "Authorization: Bearer <your-token>" \
-H "Content-Type: application/json" \
-d '{
  "title": "Order #1234",
  "customer": "Jane Doe",
  "items": [
    {"name": "Shirt", "quantity": 2, "price": 24.99}
  ]
}' --output packing-slip.pdf
